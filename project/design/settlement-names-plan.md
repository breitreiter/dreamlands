# Settlement Names Plan

## Status: Paused (waiting on Revathi language rework)

## Problem

Settlement names are generated by `FlavorText.SettlementName()` in `lib/Flavor/FlavorText.cs`,
which cycles through 4 biome roots x 5 size suffixes. This produces massive duplication on
a 100x100 map (5x "Grasshollow", 7x "Thornhollow", etc). Names are baked into map.json
at mapgen time by `ContentPopulator.NameSettlements()`.

## Plan

Replace the combinatorial generator with curated name pools per biome/tier, drawn without
replacement at mapgen time. 25 names per pool, 10 pools (5 biomes x 2 tiers), 250 names total.

### Pool Flavor (from lore files)

- **Plains T1**: Archaic British village names (Bramfield, Merewick, Harthcombe, Cresswell Vale, Longmere...)
- **Plains T2**: Ruined fort / military infrastructure (Fort Redbank, Grainway Station, Dunfield Watch, Forward Depot Three...)
- **Scrub T1**: Plateau clan names -- hard consonants, compact (Gharok, Tarsin, Kethar Pass, Sarghun Ridge...)
- **Scrub T2**: Mix of clan and Kesharat administrative names (Talvek, Orast, Vornak Hollow...)
- **Forest T1**: Short, blunt forest folk names (Thornrun, Brask, Redbark, Skell Hollow...)
- **Forest T2**: Border Reiver style + calcified camp names (Burnfoot, Timonscamp, Crookholme...)
- **Mountains T1**: Appalachian terrain-based names (Briar Run, Kestrel Hollow, Ashfall, Dane's Ridge...)
- **Mountains T2**: Generic Swiss-German cultural names -- NOT institution-specific, no "Mine" or "Monastery" (Hohrthal, Kaltbach, Steinmund...)
- **Swamp T1**: Revathi names -- vowel-heavy, water references (Kelwasi, Orathven, Tidemark...) **BLOCKED on language rework**
- **Swamp T2**: Deeper Revathi + temple-adjacent names (Merenthi, Nalith Bend...) **BLOCKED on language rework**

### T3 Settlements

T3 settlements are a mapgen bug -- the settlement placer doesn't know about tiers (tiers are
assigned after placement). Fix: strip or skip settlements in T3 regions during ContentPopulator,
after TierAssigner runs. Separate change from naming.

### Implementation

1. Add name pools to `lib/Flavor/FlavorText.cs` (or a new `SettlementNames.cs` in Flavor)
2. `SettlementName()` takes biome, tier, and a Random; draws from pool without replacement
3. Fallback if pool exhausted: generate a synthetic name (current approach or similar)
4. `ContentPopulator.NameSettlements()` passes the mapgen Random through
5. Fix T3 bug separately in SettlementPlacer or ContentPopulator
